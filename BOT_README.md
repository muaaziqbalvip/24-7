## 🆕 What's new in V22

| Feature | What changed |
|---|---|
| 📈 **Trading Signal Analyzer** | Send a chart screenshot (Quotex/IQ Option/Binomo/TradingView/MT4-MT5) and the bot analyzes candles, trend, support/resistance and momentum across 4 strategies (plus RSI/MACD/Bollinger/EMA if visible), then replies with a confidence-scored UP/DOWN/WAIT signal **and** an annotated image with a big arrow drawn on the chart. |
| 🎙️ **Voice Reply (TTS)** | Toggle with `/voice`. When ON, every AI text answer is also sent back as a Telegram voice note, generated by Groq's `canopylabs/orpheus-v1-english` (Orpheus) model. *(Groq deprecated `playai-tts` on Dec 23, 2025 — this build already uses its replacement.)* |
| 🧩 **Full multi-model routing** | The bot no longer uses one Groq model for everything — it picks the right model per task: `gpt-oss-120b` for normal chat, `gpt-oss-20b` for latency-sensitive fast replies (groups, quick acks), `qwen3.6-27b` for vision/charts, `whisper-large-v3-turbo` (→ `whisper-large-v3` fallback) for voice-in, `orpheus-v1-english` for voice-out. Run `/models` to see the full table live, `/testgroq` to verify each one is reachable. |
| 👁️ **Upgraded Vision** | General photo analysis now tries **Groq Qwen3.6-27b** (fast, multimodal) first, falling back to Gemini Vision if it fails. |
| 👥 **Group Shared Memory** | Groups now have their own conversation history (`group_memory` table), tagged with each speaker's name, so the bot can follow a multi-person conversation instead of only remembering the last person who triggered it. |
| ⚡ **Model refresh** | Groq's chat model was migrated from the deprecated `llama-3.3-70b-versatile` to `openai/gpt-oss-120b` (Groq deprecated the old model in mid-2026). |

Everything from V21 (multi-AI routing, PDF/Word/ZIP generation, image generation, web search, admin panel, dashboard, register/login, rate limiting, voice transcription) is unchanged and still works.

## ⚙️ Setup

### 1. Environment variables
Set these before running (`.env` file or your host's secret manager):

```
BOT_TOKEN=your_telegram_bot_token
GEMINI_API_KEY=your_gemini_key
GROQ_API_KEY=your_groq_key
OPENROUTER_KEY=your_openrouter_key
ADMIN_ID=your_telegram_numeric_user_id
GOOGLE_API_KEY=          # optional, for Google Custom Search fallback
GOOGLE_CX=               # optional, matching Search Engine ID
```

`GROQ_API_KEY` is now required for more than just Whisper — it also powers chart vision analysis (Qwen3.6-27b) and voice replies (Orpheus TTS), so keep it valid and check your Groq usage tier.

### 2. Install dependencies
```bash
pip install -r requirements.txt --break-system-packages
```

### 3. Run
```bash
python3 bot_v22.py
```

The bot auto-restarts on crash (5s delay, infinite loop) and logs to `mi_titan_v22.log`. On startup it DMs `ADMIN_ID` an "I'm online" message with the branded logo, if `logo_v22_small.jpg` is present next to the script.

## 📈 Using the Trading Signal feature

Two ways to trigger it:

1. **Explicit:** Send `/signal`, then send your chart screenshot as the next photo.
2. **Automatic:** Just send a chart screenshot with a caption containing a trading-related word (`chart`, `signal`, `trade`, `quotex`, `candle`, `forex`, etc.) — no `/signal` needed.

The bot replies with:
- A confidence bar (0–100%) and UP/DOWN/WAIT direction
- Which of the 4 built-in strategies agreed (trend-following, candlestick reversal, support/resistance, momentum)
- A short reasoning paragraph in Roman Urdu
- **An annotated copy of your chart image** with a large green (UP) / red (DOWN) / yellow (WAIT) arrow and a confidence badge drawn directly onto it

⚠️ **This is pattern-recognition on a screenshot, not financial advice.** Binary options / short-expiry trading carries very high risk of loss. The bot always includes a risk disclaimer in its signal output — please don't strip that out when customizing.

## 🎙️ Using Voice Reply

- `/voice` toggles voice replies on/off (also available as a button in `/menu`)
- When ON, the bot sends a spoken version of its text answer as a Telegram voice note (via Groq Orpheus TTS, voice: `autumn` by default — change it in `NeuralEngine.call_groq_tts`'s default argument. Other English voices: diana, hannah, austin, daniel, troy)
- Voice replies are capped at ~950 characters of speech to keep generation fast; long answers are spoken as a trimmed summary while the full text is still sent as a message

## 👥 Group behavior

- Mention the bot (`@yourbotusername`) or reply to one of its messages to get a full AI answer with group-shared memory context
- Any other message in the group is still logged into that group's shared memory (so context isn't lost) and gets a short 1–2 line reply
- Use `/clear` semantics per-group by extending `db.clear_group_history(chat_id)` if you want an admin command for it (not wired to a command by default — add one in the Admin section if needed)

## 🗄️ Database

SQLite file: `mi_titan_v22.db` (fresh file — V21's `mi_titan_v21.db` is not read automatically). If you want to preserve your old users/history, either rename your old DB file to `mi_titan_v22.db` before first run (the code safely `ALTER TABLE`s in the new columns), or write a one-off migration script.

New tables/columns vs V21:
- `users.voice_reply` (INTEGER, default 0)
- `memory.chat_id`, `memory.author` (added for forward-compatibility; general per-user memory still keys off `uid`)
- `group_memory` (new table: `chat_id`, `role`, `author`, `content`, `engine`, `ts`)

## 🩺 Troubleshooting (important — read this if voice/signal isn't working)

**Run `/testgroq` first.** It directly pings Groq's chat, vision, and TTS
models and tells you exactly which one is broken and why, instead of you
guessing.

**#1 most common issue: Groq model terms not accepted.**
Groq's newer/preview models (`qwen/qwen3.6-27b` for vision, `canopylabs/orpheus-v1-english` (Orpheus TTS)
for voice) require you to manually accept their terms **once per Groq
account** before the API key can use them — this is a Groq account
setting, not a bug in the bot. If `/testgroq` shows "terms not accepted":

1. Log into your Groq account at https://console.groq.com
2. Visit `https://console.groq.com/playground?model=qwen/qwen3.6-27b` → click **Accept Terms**
3. Visit `https://console.groq.com/playground?model=canopylabs/orpheus-v1-english` → click **Accept Terms**
4. Try `/testgroq`, `/testvoice`, or `/signal` again — no redeploy needed, it works immediately after accepting.

**Voice reply still silent after accepting terms?**
- Confirm `/voice` shows "ON 🔊" (check with the menu button too — the toggle can silently be off)
- Run `/testvoice` — it bypasses your voice_reply setting and tests TTS directly, and now reports the exact HTTP error if it fails instead of doing nothing
- Check `mi_titan_v22.log` for `Voice reply HTTP error:` lines

**Trading signal image (annotated chart) not appearing?**
- This was a real bug in the first V22 build: the image-drawing code could
  silently crash on some servers (missing bold font, or emoji glyphs that
  the fallback font can't render) and fall back to text-only *without
  telling you*. Fixed now — it logs the full traceback to
  `mi_titan_v22.log` under `draw_signal_on_chart failed:`, and the bot
  now explicitly tells you in the chat if the image couldn't be generated
  instead of staying silent about it.
- If you still see the "image nahi ban saki" warning, check the log for
  the traceback and install `fonts-dejavu-core` (`apt install
  fonts-dejavu-core` on Debian/Ubuntu-based hosts) if it's a font issue.

**Signal text looks garbled / not proper JSON?**
- Also fixed: the vision call now forces Groq's native JSON mode
  (`response_format: {"type": "json_object"}`) instead of only asking for
  JSON in the prompt, which is what Groq's own docs recommend for
  reliable structured output from `qwen/qwen3.6-27b`.
- `qwen/qwen3.6-27b` is officially a **preview model** on Groq (their
  words, not ours) — it can change behavior or be deprecated with little
  notice. If quality degrades again in the future, check
  https://console.groq.com/docs/vision for the current recommended vision
  model and swap it into `NeuralEngine.call_groq_vision`'s `"model"` field.
